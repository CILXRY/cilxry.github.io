# 📦 Blog 提示框功能使用指南

## 概述

为博客文章添加了多种提示框（Callout）功能，用于增强文章的可读性和信息层次。

## 🎯 支持的类型

| 类型 | 语法 | 图标 | 颜色 | 用途 |
|------|------|------|------|------|
| 信息 | `[info]` | ℹ️ | 蓝色 | 提供额外背景信息或说明 |
| 警告 | `[warn]` 或 `[warning]` | ⚠️ | 橙色 | 提醒用户注意某些事项 |
| 错误 | `[error]` | ❌ | 红色 | 表示操作失败或存在问题 |
| 成功 | `[success]` 或 `[tip]` | ✅ | 绿色 | 表示操作完成或提供技巧建议 |
| 注释 | `[note]` | 📝 | 灰色 | 补充说明或备注 |
| 危险 | `[danger]` | 🚫 | 深红色 | 警告严重后果 |

## 📝 使用方法

### 基础语法

在 Markdown 文件中，另起一行并使用以下格式：

```markdown
[类型] 提示框内容
```



### 支持 Markdown 格式

提示框内支持常见的 Markdown 语法：

```markdown
[info] 这是一个包含多种格式的提示框：
- 支持 **列表项**
- 支持 `行内代码`
- 支持 [链接](https://example.com)
- 支持 **粗体** 和 *斜体*
```

## 🎨 设计特性

### 响应式配色
- 浅色模式和深色模式下自动适配
- 每种类型都有独特的配色方案
- 使用 OKLCH 色彩空间确保色彩一致性

### 动画效果
- 鼠标悬停时轻微位移
- 阴影渐变效果
- 平滑过渡动画（0.3s）

### 无障碍性
- 清晰的视觉层次
- 合适的对比度
- 语义化的 HTML 结构

## 💡 使用场景

### 技术文档
- `[warn]` - 警告用户注意潜在风险
- `[tip]` - 提供实用技巧和最佳实践
- `[error]` - 说明常见错误和解决方案

### 教程指南
- `[info]` - 补充背景知识
- `[note]` - 添加额外说明
- `[success]` - 标记完成的步骤

### 安全提示
- `[danger]` - 强调危险操作
- `[warn]` - 提醒注意事项

## 🔧 技术实现

### 组件结构
- **Callout.vue** - Vue 组件，负责渲染提示框
- **remark-callout.ts** - Remark 插件，解析 Markdown 语法
- **astro.config.mjs** - Astro 配置，集成插件

### 文件位置
```
src/
├── components/
│   └── Blog/
│       └── Callout.vue          # 提示框组件
├── plugins/
│   └── remark-callout.ts        # Markdown 解析插件
└── content/
    └── blogs/
        └── 提示框功能测试.md      # 测试文章
```

## 📋 注意事项

1. **语法要求**
   - 提示框标记必须在段落开头
   - 方括号后需要跟一个空格
   - 类型名称不区分大小写

2. **样式定制**
   - 所有样式使用 CSS 变量定义
   - 可通过修改 `Callout.vue` 中的 CSS 变量来自定义配色
   - 支持暗色模式自动适配

3. **性能优化**
   - 使用 CSS transform 而非 margin/padding 实现动画
   - 避免使用昂贵的 filter 效果
   - 采用硬件加速的 transition
